## Why We Believe Things

Status: Draft
Depends On:
- RFC-0000

---

# Purpose

Knowledge without evidence is opinion.

Confidence without provenance is meaningless.

This RFC defines:

- Evidence
- Sources
- Assets
- Provenance
- Trust
- Citation
- Lineage
- Extraction

This RFC answers:

Why do we believe this?

---

# Core Principle

Evidence is not Truth.

Evidence contributes to Beliefs.

Sources do not become true because they exist.

Evidence influences confidence.

---

# Evidence Hierarchy

Reality
→ Observation
→ Evidence
→ Assertion
→ Belief
→ Fact-like Belief

Delphi stores the chain.

---

# Evidence Types

## Direct Observation

Examples:

- Benchmark Result
- Sensor Reading
- User Interview
- Experiment

Highest quality when reproducible.

---

## Primary Sources

Examples:

- Laws
- Court Decisions
- Research Papers
- Official Specifications
- Government Publications

---

## Secondary Sources

Examples:

- Books
- Reviews
- Summaries
- Commentary

Interpret primary sources.

---

## Tertiary Sources

Examples:

- Blog Posts
- Wikis
- AI Summaries

Useful for discovery.

Lower trust.

# Assets

Assets are containers of evidence.

Examples:

- PDF
- Video
- Audio
- Repository
- Presentation
- Web Page
- Dataset
- Meeting Recording
- Email

Assets are leaves.

---

# Asset Schema

```ts
interface Asset {
  id: string

  assetType:
    | "PDF"
    | "VIDEO"
    | "AUDIO"
    | "WEBPAGE"
    | "REPOSITORY"
    | "DATASET"
    | "EMAIL"
    | "PRESENTATION"

  title: string

  author?: string

  publishedAt?: string

  sourceUrl?: string
}
```

---

# Passages

Assets contain passages.

Examples:

- Paragraph
- Section
- Timestamp Range
- Code Snippet
- Table

Passages are the smallest citation unit.

---

# Passage Schema

```ts
interface Passage {
  assetId: string

  location: {
    page?: number
    section?: string
    timestamp?: string

    charStart?: number
    charEnd?: number
  }

  text?: string
}
```

---

# Evidence References

Evidence links beliefs to sources.

```ts
interface EvidenceRef {
  sourceLeafId: string

  relation:
    | "SUPPORTS"
    | "CONTRADICTS"
    | "MENTIONS"
    | "INTERPRETS"
    | "DERIVES_FROM"

  strength: number

  extractionConfidence: number

  passage?: Passage
}
```

---

# Provenance

Every leaf must answer:

- Who created it?
- How was it created?
- What was used?
- When was it created?
- Who reviewed it?

---

# Provenance Schema

```ts
interface Provenance {
  createdBy: AgentRef

  createdAt: string

  generatedBy?: ActivityRef

  derivedFrom: string[]

  importedFrom?: ImportRef

  reviewedBy?: AgentRef[]

  lastReviewedAt?: string
}
```

---

# Agent References

```ts
interface AgentRef {
  id: string

  type:
    | "HUMAN"
    | "AGENT"
    | "SYSTEM"
    | "IMPORT"

  name?: string
}
```

```ts
interface ActivityRef {

    | "EVALUATION"
    | "IMPORT"
    | "MERGE"
    | "MIGRATION"

```

# Source Reliability

Not all sources are equal.

Reliability should be modeled.

Examples:

Official Specification
0.95

Peer Reviewed Paper
0.90

Government Publication
0.85

Industry Benchmark
0.80

Expert Interview
0.75

Blog Post
0.40

AI Generated Summary
0.25

---

# Source Diversity

Confidence increases when:

Multiple independent sources agree.

Example:

Single Source
→ Low diversity

Ten independent sources
→ High diversity

---

# Contradictory Evidence

Contradictions must be preserved.

Example:

Paper A supports claim.

Paper B contradicts claim.

Both remain visible.

---

# Evidence Quality

Every evidence reference should have:

```ts
interface EvidenceQuality {
  reliability: number

  diversity: number

  freshness: number

  reproducibility: number

  traceability: number
}
```

---

# Human vs Agent Evidence

Evidence generated by agents must be marked.

Example:

Human Observation

Agent Extraction

Agent Summary

Agent Interpretation

Agent-generated evidence should never masquerade as primary evidence.

# Missing Evidence

A belief without evidence creates Knowledge Debt.

Examples:

Unsupported Assumption

Missing Citation

Missing Benchmark

Missing Research

These should generate tasks.

---

# Citation Rules

All beliefs should eventually support:

At least one citation.

All citations should reference:

- Asset
- Passage
- Relationship Type

---

# Research Integration

Research consumes evidence.

Research produces:

- New Evidence
- New Beliefs
- New Questions

---

# Canonical Questions

Every evidence object should answer:

Where did this come from?

Who created it?

Who reviewed it?

What supports it?

What contradicts it?

How reliable is it?

How fresh is it?

Can it be reproduced?

---

# Success Criteria

A Delphi system successfully implements this RFC when:

1. Every belief has provenance.
2. Every belief can be traced to evidence.
3. Contradictory evidence is preserved.
4. Source reliability is measurable.
5. Evidence lineage is visible.
6. Missing evidence creates research tasks.
7. Agent-generated evidence is clearly marked.
8. Citations are first-class objects.
9. Evidence quality is measurable.
10. Knowledge can explain itself.
