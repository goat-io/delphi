# RFC-0000 — Delphi Constitution

## Purpose

Delphi is an Agent Knowledge & Decision Protocol.

Its purpose is not to store information.
Its purpose is to continuously improve a model of reality through evidence, reasoning, evaluation, decision-making, execution, and learning.

Core questions:

- Why do we believe this?
- What supports it?
- What contradicts it?
- How confident are we?
- What depends on it?
- What breaks if it becomes false?
- How should it be evaluated?
- What should be researched next?

---

# First Principles

1. Reality is not Knowledge.
2. Evidence is not Truth.
3. Knowledge is not certain.
4. Evaluation is separate from Knowledge.
5. Decisions are separate from Beliefs.
6. Ontology is Knowledge.
7. Everything evolves.
8. Every assertion requires provenance.
9. Every belief requires confidence.
10. Every system accumulates knowledge debt.

---

# Agent Loop

Reality
→ Evidence
→ Beliefs
→ Evaluation
→ Decisions
→ Tasks
→ Execution
→ Observation
→ Reality

This loop is the heart of Delphi.

---

# Core Primitives

## Assertion

The atomic unit of reasoning.

Examples:

- TigerBeetle is suitable for Walliver.
- Roman law influenced Chilean law.
- Accessibility improves usability.

Assertions can have:

- Evidence
- Confidence
- Dependencies
- Evaluations
- Decisions
- Tasks

## Leaf

Canonical storage object.

Everything is stored as a leaf.

Examples:

- Object
- Evidence
- Belief
- Decision
- Rubric
- Task
- Ontology Type

## Edge

Relationship between leaves.

## Evidence

Supporting or contradicting information.

## Confidence

Quantified belief state.

## Rubric

Structured evaluation model.

## Task

Future work.

## Ontology Object

Type, relationship, validation rule, or ontology migration.

---

# The Seven Layers

## Reality Layer

Things that exist.

Examples:

- PostgreSQL
- Goatlab
- GDPR

## Evidence Layer

Things that describe reality.

Examples:

- Books
- Papers
- Laws
- Interviews

## Knowledge Layer

Interpretations of evidence.

Examples:

- Claims
- Beliefs
- Assumptions
- Hypotheses

## Evaluation Layer

Quality standards.

Examples:

- Design rubric
- Architecture rubric
- Legal rubric

## Decision Layer

Chosen actions.

Examples:

- Use TigerBeetle
- Adopt OAuth2

## Research Layer

Mechanisms that reduce uncertainty.

Examples:

- Questions
- Experiments
- Investigations

## Ontology Layer

The language of the system.

Examples:

- Types
- Relationships
- Validation rules

---

# Knowledge Dependency Theory

Knowledge is built upon knowledge.

Examples:

Arithmetic
→ Algebra
→ Linear Algebra
→ Machine Learning

Roman Law
→ Civil Law
→ Chilean Civil Code

Every leaf must answer:

- What do I depend on?
- What depends on me?

Impact analysis is a first-class capability.

---

# Confidence Theory

Truth is not binary.

Confidence is represented as:

- Evidence Strength
- Source Reliability
- Source Diversity
- Freshness
- Consensus
- Contradiction Risk

Facts are high-confidence beliefs.

---

# Evaluation Theory

Knowledge tells us what we believe.

Evaluation tells us what good looks like.

Rubrics are first-class objects.

Rubrics contain:

- Criteria
- Weights
- Scoring rules
- Version history
- Evidence
- Provenance

---

# Ontology Theory

Ontology is not hardcoded.

Ontology is represented using leaves.

Ontology can:

- Evolve
- Be researched
- Be evaluated
- Be versioned

Ontology changes occur through:

Gap
→ Proposal
→ Research
→ Review
→ Migration

---

# Research Theory

Research exists to reduce uncertainty.

Research loop:

Question
→ Investigation
→ Evidence
→ Belief Update
→ Gap Detection
→ New Questions

The process never ends.

---

# Knowledge Sovereignty

Brains own:

- Knowledge
- Evidence
- Confidence
- Ontology extensions
- Rubrics
- Decisions

Brains share protocol.

Brains do not share ownership.

---

# Federation

Brains may:

- Import
- Export
- Reference
- Fork
- Subscribe

without losing sovereignty.

---

# Canonical Rules

1. Everything is a Leaf.
2. Leaves are canonical.
3. Indexes are generated.
4. Documents are not truth.
5. Evidence supports assertions.
6. Beliefs have confidence.
7. Rubrics evaluate quality.
8. Ontology evolves.
9. Research never ends.
10. Every belief must be explainable.

---

# Long-Term Vision

Delphi becomes a portable protocol for:

- Knowledge
- Evaluation
- Decision Making
- Research
- Ontology Evolution

The goal is not to answer questions.

The goal is to continuously improve a model of reality.
