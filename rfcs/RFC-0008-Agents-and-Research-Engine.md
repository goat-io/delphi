# Purpose

Knowledge that does not evolve becomes stale.

Delphi is designed around continuous learning.

Agents do not exist to answer questions.

Agents exist to improve Brains.

This RFC defines:

- Agent Architecture
- Research Loops
- Navigation Loops
- Gap Detection
- Knowledge Debt Processing
- Continuous Learning
- Brain Evolution

---

# Core Principle

Agents should not retrieve knowledge.

Agents should navigate understanding.

The goal is not answering questions.

The goal is improving the Brain.

---

# The New Agent Model

Traditional AI:

Question
→ Search
→ Context
→ Answer

Delphi:

Question
→ Index
→ Navigation
→ Knowledge
→ Evidence
→ Evaluation
→ Answer

Understanding precedes retrieval.

---

# The Eternal Loop

Observation
→ Question
→ Navigation
→ Research
→ Evidence
→ Belief Update
→ Evaluation
→ Gap Detection
→ New Questions
→ Repeat

The loop never ends.

---

# Agent Philosophy

Agents do not own knowledge.

Brains own knowledge.

Agents temporarily operate on Brains.

This prevents memory fragmentation.

---

# The Navigation Principle

Before reading any leaf:

Read the Index.

Before reading any evidence:

Read the Topic Index.

Before researching:

Read the Research Index.

Navigation first.

Retrieval second.

---

# Agent Taxonomy

Delphi defines specialized agents.

Agents may be:

- LLMs
- Workflows
- Humans
- Hybrid Systems

---

# Navigator Agent

Purpose:

Find the best path through knowledge.

Responsibilities:

- Read indexes
- Build navigation plans
- Discover related topics
- Minimize context usage

Outputs:

- Navigation Paths
- Recommended Reading Order

---

# Research Agent

Purpose:

Acquire knowledge.

Responsibilities:

- Search sources
- Gather evidence
- Generate findings
- Create research leaves

Inputs:

- Questions
- Gaps
- Tasks

Outputs:

- Evidence
- Findings
- Claims

---

# Critic Agent

Purpose:

Challenge beliefs.

Responsibilities:

- Find contradictions
- Detect assumptions
- Challenge confidence
- Identify weak evidence

Outputs:

- Contradictions
- Research Tasks
- Confidence Adjustments

---

# Synthesizer Agent

Purpose:

Create understanding.

Responsibilities:

- Merge findings
- Update beliefs
- Generate indexes
- Produce summaries

Outputs:

- Beliefs
- Knowledge Updates
- Index Updates

---

# Evaluator Agent

Purpose:

Apply rubrics.

Responsibilities:

- Score outputs
- Measure quality
- Recommend improvements

Outputs:

- Evaluation Results
- Quality Reports

---

# Ontology Steward

Purpose:

Maintain ontology health.

Responsibilities:

- Detect gaps
- Merge duplicates
- Propose new types
- Propose new relationships

Outputs:

- Ontology Tasks
- Ontology Migrations

---

# Auditor Agent

Purpose:

Ensure trustworthiness.

Responsibilities:

- Verify provenance
- Verify citations
- Detect stale knowledge
- Detect missing evidence

Outputs:

- Audit Reports

---

# Planner Agent

Purpose:

Convert uncertainty into work.

Responsibilities:

- Generate tasks
- Prioritize research
- Allocate resources

Outputs:

- Action Plans
- Research Backlogs

---

# Knowledge Debt Processor

Purpose:

Reduce uncertainty.

Inputs:

- Low confidence beliefs
- Missing evidence
- Missing evaluations
- Ontology gaps
- Stale knowledge

Outputs:

- Research backlog

# Canonical Rules

1. Agents do not own knowledge.
2. Brains own knowledge.
3. Navigation precedes retrieval.
4. Questions are first-class objects.
5. Research generates evidence.
6. Evidence updates beliefs.
7. Beliefs update indexes.
8. Indexes guide navigation.
9. Knowledge debt creates work.
10. Every agent action must be auditable.

---

# Success Criteria

1. Questions are first-class leaves.
2. Research generates evidence.
3. Evidence updates beliefs.
4. Indexes update automatically.
5. Agents navigate before retrieving.
6. Knowledge debt is measurable.
7. Navigation debt is measurable.
8. Tasks are generated automatically.
9. Agent collaboration is reproducible.
10. The Brain continuously improves over time.
